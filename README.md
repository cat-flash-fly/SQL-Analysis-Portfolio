## 🎯 分析目标（与我 Tableau 看板的四个维度对应）

我在 Tableau Public 中已发布交互式看板，涵盖以下四个核心分析维度。  
本仓库中的 SQL 脚本正是为生成这些看板提供数据准备和逻辑验证。

| 维度 | 分析内容 | 关键指标 |
| :--- | :--- | :--- |
| **1. 年度销售总览** | 按年份汇总整体销售表现 | 总销售额、总销量、客单价（每箱均价）、交易笔数 |
| **2. 国家维度分析** | 按年份+国家汇总，并计算各指标的**同比环比** | 销售额、销量、客单价及各自增长率 |
| **3. 产品维度分析** | 按年份+产品汇总，并计算各指标的**同比环比** | 同上，识别明星产品与衰退产品 |
| **4. 月度成交趋势** | 按月汇总，观察季节性波动 | 月度销售额、销量、订单数、月度环比增长率 |

**额外增值分析**（SQL 专属，未在 Tableau 中呈现）：
- 每个国家销售额 Top 3 畅销产品（`RANK() OVER(PARTITION BY ...)`）
- 国家市场价值分层（`CASE WHEN` 高/中/低价值市场）
- 月度同比分析（2023 vs 2022 同月对比）

## 🛠️ SQL 技术亮点（体现我的 MySQL 能力）

本项目的 SQL 脚本展示了数据分析师必备的以下技能：

| 技术点 | 应用场景 | 示例代码片段 |
| :--- | :--- | :--- |
| **窗口函数 – LAG()** | 计算环比/同比增长率 | `LAG(SUM(amount), 1) OVER (PARTITION BY country ORDER BY year)` |
| **窗口函数 – RANK()** | 分组 Top N 分析 | `RANK() OVER (PARTITION BY country ORDER BY SUM(amount) DESC)` |
| **窗口函数 – ROW_NUMBER()** | 数据去重（保留最新记录） | `ROW_NUMBER() OVER (PARTITION BY ... ORDER BY ...)` |
| **CTE（公用表表达式）** | 拆解复杂查询，提升可读性 | `WITH country_yearly AS (...)` |
| **CASE WHEN** | 业务规则分类（客户/市场分层） | `CASE WHEN SUM(amount) >= 500000 THEN '高价值' ... END` |
| **NULLIF** | 避免除零错误 | `SUM(amount) / NULLIF(SUM(boxes_shipped), 0)` |
| **数据清洗** | 金额字符转数值、日期解析、空值检查 | `REPLACE(REPLACE(amount,'$',''),',','')` |
